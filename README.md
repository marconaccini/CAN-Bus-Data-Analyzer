# CANBusLogs_2_CSV + CSV Data Analyzer

**CANBusLogs_2_CSV** is a Python converter that transforms CAN bus log files into a **signal-level CSV** by decoding frames with one or more **DBC** databases. A companion **CSV Data Analyzer** (HTML/JS/CSS) is included to inspect and search the converted data locally in a browser.

This repository contains two main parts:

- **Converter** (Python)
  - `CANBusLogs_2_CSV.py`: CLI converter
  - `CANBusLogs_2_CSV_GUI.py`: Tkinter GUI wrapper
- **Analyzer** (Browser)
  - `CAN Bus Data Analyzer.html`
  - `script.js`
  - `style.css`

---

## Features

### Converter (Python)

- **Multi-format CAN log parsing** (single entry point)
  - BusMaster 3.2.2
  - PCAN-View 2.x and 4.x
  - CL2000
  - TRC / text logs (depending on the logger)
  
- **DBC-based decoding**
  - Physical values with scaling/offset
  - Optional extraction of `VAL_` symbolic decoding into an additional column (see `--extract_values`) 

- **CSV output options**
  - Custom delimiter (default `;`)
  - Signal naming mode: `signal` or `message.signal`
  - Optional **message counter** and **message pulser** pseudo-signals
  - Optional **units row** (recommended for compatibility with the Analyzer)

- **MDF4 support in the GUI**
  - The GUI can detect `.mf4` and convert it to `.trc` using `mdf2peak.exe` shipped under `_aux/` before running the Python converter.

### CSV Data Analyzer (Browser)

A lightweight, offline analyzer that works on the CSV produced by the converter.

- **Local file load** (no server): the CSV is read with `FileReader` directly in the browser.
- **Signal list + wildcard filter** using `*` and `?`.
- **Selection table** showing (per current sample): signal name, physical value, raw value, unit.
- **Timeline navigation**
  - Slider across samples
  - Previous/Next step
  - Direct index jump
- **Chart view** (Chart.js via CDN)
  - Plots the selected signal if numeric
  - Vertical cursor line at the current sample index
- **Event search** on a selected signal
  - Value change
  - Equal / greater / less than a threshold

## Supported input formats (converter)

The converter is designed to parse log lines generated by common tools. Examples are documented in the script header.

## CSV format expectations (Analyzer compatibility)

The Analyzer expects a **semicolon-separated** CSV (default for the converter) and supports an IBA-like layout:

1. **Header row** with signal names (first column is time).
2. **Optional units row** (the Analyzer detects it if the first cell is `sec`).
3. Data rows.

**Recommendation:** to use the Analyzer with the converter output, keep the delimiter as `;` and enable the units row (`--units` or GUI option).

## Usage

### 1) Convert CAN logs to CSV (CLI)

Basic usage:

```bash
python CANBusLogs_2_CSV.py <log_file> <dbc1> [dbc2 ...] -o output.csv
```

Common options (as used by the GUI):

- `-o <file>`: output CSV path
- `-d <char>`: CSV delimiter (default `;`)
- `-n <mode>`: signal naming mode: `signal` or `message.signal`
- `--message_counter`: add message counter pseudo-signal
- `--message_pulser`: add message pulser pseudo-signal
- `--units`: add units row
- `--extract_values`: decode `VAL_` symbolic values into extra columns

Example:

```bash
python CANBusLogs_2_CSV.py run.log MCU_TM4.dbc TRK_CAN1.dbc \
  -o out.csv -d ";" -n message.signal --units --extract_values
```

### 2) Convert CAN logs to CSV (GUI)

Run:

```bash
python CANBusLogs_2_CSV_GUI.py
```

Workflow:

1. Select **log file** (CAN log / TRC / MDF4 `.mf4`).
2. Add one or more **DBC** files.
3. Choose output CSV path and options.
4. Click **Convert!**

If an `.mf4` is selected, the GUI converts it to `.trc` via `_aux/mdf2peak.exe` and then runs the Python converter.

### 3) Analyze the generated CSV (Browser)

Open the analyzer page:

- Double-click `CAN Bus Data Analyzer.html`, or
- Serve the folder with a local HTTP server (recommended for stricter browser policies):

```bash
# from the analyzer folder
python -m http.server 8000
```

Then open:

```text
http://localhost:8000/
```

Steps:

1. Click **Select CSV File** and load the CSV generated by the converter.
2. Use the left filter box to find signals (`*` and `?` wildcards).
3. Click signals to select them for the table; select one numeric signal to plot.
4. Navigate samples with the slider / Prev / Next / direct index.
5. Use **Event search** to jump to changes or threshold conditions.

---

## Notes and limitations

- The Analyzer is designed around a semicolon-delimited CSV and optionally reads the units row when the first cell is `sec`. If you change delimiter or disable units, you may need to adapt `script.js` accordingly.
- The Analyzer will not plot non-numeric signals; it ignores them for charting.
- MDF4 conversion relies on an external executable (`mdf2peak.exe`) and is currently integrated in the GUI workflow.
